Extract fragment/ package with core types for detect/sources decoupling

Move Fragment, CommitInfo, RemoteInfo, Source, FragmentsFunc, and
InnerPathSeparator from sources/ into a new fragment/ package.
Type aliases in sources/ preserve backwards compatibility.

This is the first step toward allowing detect/ to be imported as a
library without compiling the heavy sources/ dependencies (archives,
gitdiff, filetype).

Switch report/ to import fragment/ instead of sources/

Change report/finding.go to reference fragment.Fragment instead of
sources.Fragment. Since sources.Fragment is already a type alias for
fragment.Fragment, this is type-identical and breaks no callers.

Breaks the detect/ → report/ → sources/ → archives transitive chain.

Move printFinding display logic from detect/ to report/

Add PrintFinding callback field on Detector struct so detect/ no longer
needs to directly call lipgloss-dependent display functions. CLI commands
set the callback when creating a Detector. detect/ collects findings
silently when no callback is set (library usage path).

Exports printFinding as PrintFinding in report/ for cmd/ to reference.

Remove direct viper import from detect/ via config.DefaultConfigTranslated

Add config.DefaultConfigTranslated() helper that returns the translated
config without requiring detect/ to call viper directly. The full
config/types split (separating data structs from viper loading) is left
as future work — viper still reaches detect/ transitively through config/.

Remove sources import from detect/ — use fragment.Source and commit count check

Replace sources.Source parameter with fragment.Source in DetectSource.
Replace source.(*sources.Git) type assertion with len(d.commitMap) > 0
check — functionally equivalent since commitMap is only populated when
scanning git sources.

detect/ no longer imports sources/ directly. Heavy deps eliminated from
detect/ import path: archives, gitdiff, filetype, lipgloss.
